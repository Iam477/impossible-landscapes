# Foundry import notes

## Common vs chapter vs Campaign State
- **SHARED / Common**: campaign-wide concepts, recurring places, recurring cast, recurring entities, recurring tomes.
- **OP1 / Chapter**: Part One-only operation framing, Macallistar locations, tenant dossiers, clue handouts, Night Floors structure, and Part One resolution docs.
- **STATE / Campaign State**: session logs, clue reveal tracking, corruption tracking, live investigation boards, NPC status changes, permission state, and any table-specific improvisation.

## Spoiler warnings
- `OP1 - Concept - Fate of Abigail Wright [GM]` must remain GM-only.
- `OP1 - Event - Resolution of The Night Floors [GM]` must remain GM-only.
- `OP1 - Location - The Night Floors [GM]` and most night-side resident docs should not be imported to the world too early.
- `SHARED - Location - Hotel Broadalbin [GM]` should not be overexplained during Part One.

## Duplication warnings
- Do not duplicate `SHARED - Tome - The King in Yellow [GM]` inside OP1.
- Do not create separate OP1 duplicates of `The Bookshop`, `Hotel Broadalbin`, or `Mark Roark`.
- Do not store clue-state or live-play notes inside packaged canon docs.
- Keep the Night Floors as one parent location journal rather than fragmenting it into unstable room-by-room canon docs.

## Import order
1. SHARED concepts, locations, and recurring cast used by OP1.
2. OP1 briefing/framing docs.
3. Macallistar core locations and Wright case NPCs.
4. Tenant NPC journals and tenant location journals.
5. Core clue handouts.
6. OP1 concept/rules journals.
7. Night Floors locations and night-side resident journals.
8. Scenes/boards only if actively useful.

## What should stay world-only
- Session logs
- Revealed-handout permissions
- Corruption tracking
- Investigation boards altered by play
- NPC status changes caused by actual sessions
- Any live-world divergence from canon
